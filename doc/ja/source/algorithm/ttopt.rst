テンソル列最適化 ``ttopt``
***************************

``ttopt`` は、目的関数をパラメータで添字付けられた大きなテンソルとしてモデル化し、その大きなテンソルを1次元的にならんだ3階テンソルのネットワーク（行列積状態MPS, tensor train）として表現し、そのMPSを反復的な交差近似に基づく勾配不要の最適化手法で探索する ``Algorithm`` です。

前準備
~~~~~~

MPI 並列化を用いる場合は `mpi4py <https://mpi4py.readthedocs.io/en/stable/>`_ をインストールしてください。

.. code-block::

  $ python3 -m pip install mpi4py

入力パラメータ
~~~~~~~~~~~~~~

[``algorithm.param``] セクション
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- ``min_list``

  形式: 実数のリスト。長さは ``dimension`` の値と一致させます。

  説明: 各パラメータが取りうる最小値。

- ``max_list``

  形式: 実数のリスト。長さは ``dimension`` の値と一致させます。

  説明: 各パラメータが取りうる最大値。

[``algorithm.ttopt``] セクション
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

次のハイパーパラメータを指定できます。

- ``p_points``

  形式: 整数または整数のリスト。長さは ``dimension`` の値と一致させます。

  説明: パラメータ :math:`x_i` を離散化する際のボンド次元 :math:`P_i` 。
  パラメータは :math:`N_i = P_i^{q_i}` 個の等間隔点に離散化され、そのインデックスは :math:`P_i` 個の値をとる :math:`q_i` 個のテンソル脚によって表現されます。
  整数の場合は全ての次元で同じ値になります。
  デフォルトは 2 です。

- ``q_points``

  形式: 整数または整数のリスト。長さは ``dimension`` の値と一致させます。

  説明: パラメータ :math:`x_i` を離散化する際のボンド数 :math:`q_i` 。
  パラメータは :math:`N_i = P_i^{q_i}` 個の等間隔点に離散化され、そのインデックスは :math:`P_i` 個の値をとる :math:`q_i` 個のテンソル脚によって表現されます。
  整数の場合は全ての次元で同じ値になります。
  デフォルトは 1 です。

- ``r_max``

  形式: 整数 (default: 4)

  説明: 小テンソル同士をつなぐボンド次元の最大値。大きいほど表現能力はあがりますが、最適化の各ステップでコスト関数の評価回数が増えます。

- ``max_f_eval``

  形式: 整数 (default: 10000)

  説明: コスト関数の評価回数の上限。最適化の計算予算に相当します。カウンタはアルゴリズムの各スイープの終了時に更新され、この値以上になった時点でパラメータ探索を終了します。

- ``maxvol_tol``

  形式: 浮動小数点数 (default: 1.001)

  説明: 最大体積部分行列を求める際の許容誤差。許容誤差を 1 に設定すると行列の厳密な分解を行います。1 未満の値は 1 にクリップされます。

- ``maxvol_max_it``

  形式: 整数 (default: 1000)

  説明: 最大体積部分行列を求める反復の最大回数。

- ``init_points``

  形式: 実数のリストのリスト (default: [])

  説明: 最適化の冒頭で評価する初期候補。内側の各リストの長さは ``dimension`` と一致させます。任意のパラメータで、既知の候補領域を最適化器に伝えるために用います。

- ``save_eval_history``

  形式: 真偽値 (default: ``true``)

  説明: ``true`` のとき、評価した各候補点を ``ttopt_eval_history.txt`` に追記します（MPI ランク 0 のみ）。

- ``eval_history_buffer_rows``

  形式: 整数 (default: 256)

  説明: ``ttopt_eval_history.txt`` に書き出す頻度 :math:`N_{\mathrm{flush}}` 。 :math:`N_{\mathrm{flush}}` 回の評価結果をまとめて出力します。

出力ファイル
~~~~~~~~~~~~

``[base]`` の ``output_dir`` を ``OUTPUT`` と書くことにします（``root_dir`` からの相対パスで指定した場合は ``root_dir/output_dir`` の絶対パスが使われます）。
各 MPI ランクは、作業ディレクトリとして ``OUTPUT`` 直下のランク番号付きサブディレクトリ（例 ``OUTPUT/0/``）を用います。TTOpt が書き出す主なファイルは ``OUTPUT`` 直下に置かれます（下記、特に記載がなければ MPI ランク 0 のみが作成）。

``ttopt_hyperparameters.txt``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

準備フェーズ（``prepare``）の終了時に、入力で与えた主なハイパーパラメータを 1 行 1 項目で保存します。

.. code-block::

    nprocs = 1
    bounds = [[-6.0, 6.0], [-6.0, 6.0]]
    p_points = [2 2]
    q_points = [20 20]
    r_max = 4
    max_f_eval = 10000
    maxvol_tol = 1.001
    maxvol_max_it = 1000
    save_eval_history = True
    eval_history_buffer_rows = 256

``ttopt_history.txt``
^^^^^^^^^^^^^^^^^^^^^

最適化ループの各ステップ（ある MPS コア位置で候補点の束を評価し、これまでの最良点を更新した直後）ごとに、累積の関数評価回数と、その時点での最良点 ``x_opt`` および最良値 ``fx_opt`` が追記されます。先頭の ``#`` 行は列の意味です。

.. code-block::

    # $1: count
    # $2: x_opt[0]
    # $3: x_opt[1]
    # $4: fx_opt
    8 3.420030040769616e+00 -9.735097632501253e-01 7.005409201321578e+00
    24 3.420030040769616e+00 -2.098510836134754e+00 2.643948442798083e+00
    ...

次元が :math:`D` のとき、データ列は 1 列目が評価回数、2 列目から :math:`D+1` 列目が ``x_opt[0], ..., x_opt[D-1]``、最後の列が ``fx_opt`` です。

``res.txt``
^^^^^^^^^^^

終了後の全体最良解（MPI 使用時は全ランクの結果のうち最良）を、目的関数値 ``fx`` と各パラメータ（既定のラベルは ``x1``, ``x2``, …）でテキスト保存します。

.. code-block::

    fx = 3.188892404355571e-08
    x1 = 3.584424576210571
    x2 = -1.8480795365138398

``ttopt_eval_history.txt``
^^^^^^^^^^^^^^^^^^^^^^^^^^

``save_eval_history`` が ``True`` のときのみ ``OUTPUT/ttopt_eval_history.txt`` に作成されます。バッチ評価で実際に呼び出した各候補点の座標と ``f(x)`` が行単位で並びます（先頭の ``#`` 行は列ラベル。パラメータ名は ``label_list`` が設定されていればそれが使われます）。

.. code-block::

    # $1: row index (order of batches in the run)
    # $2: x1
    # $3: x2
    # $4: f(x)
    1 3.420030040769616e+00 -5.098513697160432e+00 5.218032809779967e+02
    2 3.420030040769616e+00 -9.735097632501253e-01 7.005409201321578e+00
    ...

``time.log``
^^^^^^^^^^^^

アルゴリズム全体の計測時間は、ランクごとに ``OUTPUT/<ランク番号>/time.log`` に書き出されます。

アルゴリズムの説明
~~~~~~~~~~~~~~~~~~

Tensor Train Optimization (TTOpt) [1] は、離散関数の最適値と、そのパラメータ空間上の位置を求める手法です。連続最適化問題にも容易に適用でき、高次元問題や、離散量と連続量が混在するパラメータをもつ関数にも対応できます。

TTOpt は、パラメータの組 :math:`(x_1, x_2, ..., x_n)` で添字付けられた目的関数 :math:`f(x_1, x_2, ..., x_n)` の値に対応する要素をもつ大きなテンソルの、反復的な交差近似に基づく勾配不要の最適化手法です。交差近似は近似最大体積部分行列を用いて計算されます。
各次元のパラメータ :math:`x_i` はそれぞれ :math:`N_i = P_i^{q_i}` 個の等間隔点に離散化され、そのインデックスは :math:`P_i` 個の値をとる :math:`q_i` 本のテンソル脚によって表現されます。こうして、目的関数が :math:`\prod_k q_i` 階のテンソルとして表現されます。
TTOpt アルゴリズムは、この高次元テンソルを行列積状態(MPS)として、すなわち :math:`\prod_k q_i` 個の3階テンソルの1次元ネットワークとして分解します。
巨大テンソル全体のごく一部だけを明示的に計算すればよいように設計されており、コスト関数の計算コストが高い場合や探索空間が非常に大きい場合に有利です。また、データを MPS 形式で表現することで、最適化過程で指数的に大きな行列を形成する必要を避けられます。

本アルゴリズムは複数の MPI プロセスによる実行が可能です。MPI を用いる場合、サンプル点におけるコスト関数値の評価が各ランクに分散されます。
